# 识图与读图

拿到一张技术图，如何快速看懂它在说什么。

## 三步读图法 (通用)

### 第一步：定类型

先判断这是什么图，不同类型有不同读法。

| 看到什么 | 这是什么图 | 读法 |
|----------|-----------|------|
| 方框+连线+箭头 | **流程图** | 顺着箭头走，看分支 |
| 三层/多层方框 | **架构图** | 从上到下，每层干什么 |
| 矩形框分三格 | **类图** | 看类名+关系线 |
| 竖线+横箭头 | **时序图** | 从上往下按时间读 |
| 椭圆+火柴人 | **用例图** | 火柴人=用户，椭圆=功能 |
| 圆角矩形+箭头 | **状态图** | 跟着箭头看状态流转 |
| 表格+连线+1/N | **ER 图** | 看实体和关系 |
| 圆圈+节点 | **网络拓扑** | 找核心节点+连接 |

### 第二步：找核心

每张图都有一个「主角」：

- 架构图 → 最上面/最中心的模块
- 流程图 → 从哪里开始 (起点)
- 类图 → 被最多箭头指向的类
- 时序图 → 最左边是谁先发起

### 第三步：追关系

- **线** = 有联系
- **箭头** = 有方向 (谁依赖谁、数据从哪到哪)
- **粗细** = 重要程度
- **颜色** = 区分不同层/不同系统
- **数字/标签** = 关系约束 (1对多、多对多)

## 常见图读法详解

### 1. 架构图

```
┌─────────────────────────────────────┐
│           前端层 (Frontend)          │  ← 用户看到的部分
│   React / Vue / Mobile App          │
├─────────────────────────────────────┤
│           API 网关 (Gateway)         │  ← 统一入口
├─────────────────────────────────────┤
│           业务层 (Backend)           │  ← 核心逻辑
│   ┌──────┐  ┌──────┐  ┌──────┐     │
│   │用户服务│  │订单服务│  │商品服务│     │
│   └──────┘  └──────┘  └──────┘     │
├─────────────────────────────────────┤
│           数据层 (Data)              │  ← 持久化
│   MySQL / Redis / Elasticsearch     │
└─────────────────────────────────────┘
```

**读法**：
- **从上到下** = 请求流，用户 → 前端 → 后端 → 数据库
- **同一层内** = 水平切分，服务之间有箭头表示互相调用
- **层与层之间** = 依赖方向 (上层依赖下层，不能反过来)

### 2. 流程图

```
     ┌──────┐
     │ 开始  │
     └──┬───┘
        │
     ┌──▼───┐
     │ 条件？│
     └──┬───┘
       / \
     是/   \否
     /      \
┌───▼───┐ ┌──▼───┐
│ 执行A  │ │ 执行B │
└───┬───┘ └──┬───┘
    │        │
    └───┬────┘
        │
    ┌───▼───┐
    │  结束  │
    └───────┘
```

**读法**：
- **顺着箭头走**，一条路径 = 一种情况
- **菱形** = 判断，会分叉
- **矩形** = 操作步骤
- **汇合点** = 多条路径合并，继续往下

### 3. 类图 (快速识别关系)

```
    线型速查：
    实线 △ = 继承 (is-a)        —— Dog is Animal
    虚线 △ = 实现 (implements)  —— Bird implements Flyable
    实线 ◆ = 组合 (part-of)     —— Room is part of House
    实线 ◇ = 聚合 (has-a)       —— Student has Teacher
    普通实线 = 关联              —— User has Orders
    虚线箭头 = 依赖               —— Service uses Repository
```

### 4. ER 图 (数据库)

```
┌──────────┐         ┌──────────┐
│  Student  │ 1     * │  Course  │
│──────────│─────────│──────────│
│ id (PK)  │         │ id (PK)  │
│ name     │         │ title    │
└──────────┘         └──────────┘
```

**读法**：
- **方框** = 一张表 → 对应一个实体
- **第一行** = 主键 (PK)，唯一标识
- **连线上的数字** = 关系: `1:1` 一对一, `1:N` 一对多, `M:N` 多对多
- 例子：1个学生选多门课，1门课有多名学生 → 多对多

### 5. 状态图

```
     ┌─────────┐  支付  ┌─────────┐
     │ 待支付   │──────>│ 已支付   │
     └────┬────┘       └────┬────┘
          │ 取消             │ 发货
          ▼                  ▼
     ┌─────────┐        ┌─────────┐
     │ 已取消   │        │ 已发货   │
     └─────────┘        └────┬────┘
                              │ 签收
                              ▼
                         ┌─────────┐
                         │ 已完成   │
                         └─────────┘
```

**读法**：
- **圆角框** = 一个状态
- **箭头上的字** = 触发事件
- **起点** (实心圆) → 初始状态
- **终点** (空心圆含实心) = 结束
- 跟着箭头走，就是一条状态变迁路径

## 读图速查口诀

```
先看标题定类型，再找主角和核心。
顺着箭头追逻辑，线粗色深分量重。
数字标签看关系，注释说明不遗漏。
一张图说一件事，看懂主干即成功。
```

## 常见坑

| 坑 | 怎么避 |
|----|--------|
| 图画太乱 | 先找标题，忽略装饰，只看主干 |
| 关系看不懂 | 查箭头符号表，或读图旁边标注 |
| 不知道从哪开始 | 架构图从上往下，流程图从「开始」出发 |
| 中文英文混用 | 看形状先定类型，再看文字 |
