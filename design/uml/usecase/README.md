# 用例图 (Use Case)

描述「谁能用系统做什么」。需求分析第一步。

## 基本元素

```
                    ┌─────────────┐
       ○            │  下订单      │
    用户             └──────┬──────┘
  火柴人=参与者              │
                    椭圆=功能(用例)
```

## 四种关系

### 包含 <<include>> — 必选
```
        ┌────────┐
        │ 下订单  │
        └───┬────┘
            │ <<include>>   —— 下订单必须包含支付
    ┌───────▼──────┐
    │    支付      │
    └──────────────┘
```

### 扩展 <<extend>> — 可选
```
        ┌────────┐
        │  登录   │
        └───┬────┘
            │ <<extend>>    —— 可能扩展出验证码
    ┌───────▼──────┐
    │  短信验证    │
    └──────────────┘
```

### 泛化 — 参与者继承
```
         用户
         △
        / \
    普通用户 管理员       —— 管理员也是用户,但有更多权限
```

## 实战：电商系统用例

```
                         ┌──────────┐
                    ─────│  浏览商品  │
                   /     └──────────┘
                  /
    ○           /       ┌──────────┐
  买家 ────────────────│  下订单   │
    │         /         └────┬─────┘
    │        /               │ <<include>>
    │       /         ┌──────▼─────┐
    │      /          │   支付     │
    │     /           └────────────┘
    │    /                   ↑
    │   /                    │ 继承
    │  /              ┌──────┴─────┐
    ├─────────────    │ 微信支付    │
    │                 │ 支付宝     │
    │                 └────────────┘
    │
    │         ┌──────────┐
    ├─────────│ 评价商品  │
    │         └──────────┘
    │
    │         ┌──────────┐
    └─────────│ 查看订单  │
              └──────────┘

    ○
  管理员 ──── 管理商品 / 管理订单 / 管理用户
```

**读图**：
- 先看有几个火柴人 → 几种角色
- 每个火柴人连着几个椭圆 → 这个角色能干什么
- `<<include>>` = 必须做的步骤
- `<<extend>>` = 可选的额外步骤
