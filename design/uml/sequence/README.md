# 时序图 (Sequence Diagram)

展示对象之间如何「按时间顺序」交互。最适合描述 API 调用、微服务通信。

## 基本元素

```
用户        前端        后端        数据库
 │           │           │           │
 │  点击登录  │           │           │
 │──────────>│           │           │
 │           │  POST /login          │
 │           │──────────>│           │
 │           │           │ 查询用户   │
 │           │           │──────────>│
 │           │           │   User    │
 │           │           │<──────────│
 │           │   200 OK + Token      │
 │           │<──────────│           │
 │  登录成功  │           │           │
 │<──────────│           │           │
```

## 核心符号

| 符号 | 含义 |
|------|------|
| `──>` | 同步消息 |
| `- - >` | 返回消息 |
| `──>>` | 异步消息 |
| `[X]` | 对象销毁 |
| 竖虚线 | 生命线 (对象存活) |
| 竖长条 | 激活期 (正在执行) |

## 实战：下单流程

```
用户        App        订单服务    库存服务    支付服务    消息队列
 │           │           │           │           │           │
 │  下单      │           │           │           │           │
 │──────────>│           │           │           │           │
 │           │ createOrder()         │           │           │
 │           │──────────>│           │           │           │
 │           │           │ checkStock()          │           │
 │           │           │──────────>│           │           │
 │           │           │   有货    │           │           │
 │           │           │<──────────│           │           │
 │           │           │           │ pay()     │           │
 │           │           │──────────────────────>│           │
 │           │           │           │ 支付成功   │           │
 │           │           │<──────────────────────│           │
 │           │           │           │           │ 发送消息  │
 │           │           │──────────────────────────────────>│
 │           │   订单ID   │           │           │           │
 │           │<──────────│           │           │           │
 │  下单成功  │           │           │           │           │
 │<──────────│           │           │           │           │
```

## 读图步骤

1. **找参与者** — 最上面一排，谁在交互？
2. **从上往下读** — 时间顺序，先发生的在上面
3. **看箭头方向** — 谁调用谁
4. **看返回** — 虚线返回，带结果

## 何时用

- ✅ 梳理 API 调用链
- ✅ 微服务间通信
- ✅ 登录/支付等需要多步骤的业务
- ❌ 不需要-> 用类图就够了
