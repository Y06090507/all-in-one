# 活动图 (Activity) + 状态图 (State)

## 活动图 — 业务流程

描述「一步步做什么」，可以带分支和并行。

### 基本元素
```
● = 起点        ◎ = 终点
□ = 操作步骤    ◇ = 判断分支
━━ = 分叉(并行)  ━━ = 汇合(等待)

───── 泳道 ───── = 不同角色/系统的职责区域
```

### 实战：用户注册流程

```
    用户                    系统
     │                      │
     ●                      │
     │                      │
     ▼                      │
 ┌───────┐                  │
 │ 填写表单│                  │
 └───┬───┘                  │
     │                      │
     ▼                      │
 ┌───────┐  提交        ┌────▼───┐
 │ 点击注册│─────────────>│ 验证输入 │
 └───────┘              └────┬───┘
                             │
                        ┌────◇────┐
                        │ 格式正确？│
                        └────┬────┘
                        否  /  \  是
                           /    \
                     ┌─────┘      └─────┐
                     ▼                   ▼
              ┌────────────┐      ┌────────────┐
              │ 提示错误     │      │ 检查重复     │
              └──────┬─────┘      └──────┬─────┘
                     │                   │
                     └────>回到填写表单    ◇ 重复？
                                        / \
                                      是/   \否
                                      /      \
                                ┌────┘        └────┐
                                ▼                   ▼
                         ┌──────────┐        ┌──────────┐
                         │ 提示已注册  │        │ 发送验证码  │
                         └──────────┘        └────┬─────┘
                                                  │

```

### 并行活动 (fork/join)

```
        ┌─────────────────┐
        │   订单支付成功     │
        └────────┬────────┘
                 │
        ┌────────●────────┐    ● = 分叉，同时进行
        │                  │
        ▼                  ▼
  ┌──────────┐      ┌──────────┐
  │ 扣减库存  │      │ 发送通知   │
  └────┬─────┘      └────┬─────┘
       │                  │
       └────────●────────┘    ● = 汇合，都完成才继续
                │
                ▼
        ┌──────────────┐
        │  订单状态更新  │
        └──────────────┘
```

## 状态图 — 对象生命周期

描述「一个东西从生到死经历哪些状态」。

### 核心元素

```
● = 初始状态 (实心圆)
[状态名] = 圆角矩形
  →  = 状态转换 (箭头)
事件名 = 箭头上的标注
◎ = 终态 (圆含实心)
```

### 实战：订单状态机

```
     ●
     │
     ▼
┌─────────┐
│  待支付   │
└────┬────┘
     │ 用户付款
     ▼
┌─────────┐
│  已支付   │
└────┬────┘
     │ 商家发货
     ▼
┌─────────┐
│  已发货   │
└────┬────┘
     │ 用户签收
     ▼
┌─────────┐
│  已完成   │
└────┬────┘
     │
     ◎

  旁路：
  待支付 ──超时──→ 已取消 ──→ ◎
  已支付 ──退款──→ 已退款 ──→ ◎
```

### 带守卫条件

```
        ┌─────────┐
        │  登录中   │
        └────┬────┘
             │
        ┌────◇────┐
        │ 输入次数  │
        └────┬────┘
      <3次 /  \ ≥3次
         /      \
        ▼        ▼
  ┌─────────┐ ┌─────────┐
  │ 重试     │ │ 锁定账户  │
  └────┬────┘ └─────────┘
       │ 成功
       ▼
  ┌─────────┐
  │ 已登录   │
  └─────────┘
```

## 活动图 vs 状态图 vs 流程图

| | 关注 | 适合 |
|---|------|------|
| **活动图** | 步骤流转 | 业务流程、算法 |
| **状态图** | 对象状态变化 | 订单、工单、登录 |
| **流程图** | 控制流 | 程序逻辑、if-else |
