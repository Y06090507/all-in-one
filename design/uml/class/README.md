# 类图 (Class Diagram)

UML 中最核心的图。画清楚「有哪些类」和「类之间什么关系」。

## 类的表示

```
┌─────────────────────┐
│     User            │  ← 类名
├─────────────────────┤
│ - id: int           │  ← 属性 (私有)
│ - name: String      │
│ # email: String     │  ← 受保护
│ + createdAt: Date   │  ← 公有
├─────────────────────┤
│ + login(): bool     │  ← 方法
│ + logout(): void    │
│ - hashPwd(): String │
└─────────────────────┘

符号: + public   # protected   - private   ~ package
```

## 六种关系 (最重要！)

### 1. 泛化 (继承) — 「是」的关系
```
       ┌──────────┐
       │  Animal  │
       └────┬─────┘
        △       △       △ = 空心三角箭头，指向父类
       /         \
┌──────┐      ┌──────┐
│ Dog  │      │ Cat  │
└──────┘      └──────┘
```
```python
class Animal: ...
class Dog(Animal): ...   # Dog 是 Animal
class Cat(Animal): ...
```

### 2. 实现 — 接口实现
```
┌──────────────┐
│  <<interface>>│
│   Flyable    │
└──────┬───────┘
       △ 虚线
       │
┌──────┴──────┐
│    Bird     │
└─────────────┘
```
```python
class Flyable(ABC):
    @abstractmethod
    def fly(self): ...

class Bird(Flyable):  # Bird 实现 Flyable
    def fly(self): print("flying")
```

### 3. 关联 — 「有一个」的关系 (最常用)
```
┌──────────┐  1    0..* ┌──────────┐
│   User   │────────────│   Order  │
└──────────┘            └──────────┘
  1个用户 有 0到多个订单
```
```python
class User:
    def __init__(self):
        self.orders: List[Order] = []
```

### 4. 聚合 — 「整体-部分」弱关系
```
┌──────────┐          ┌──────────┐
│  School  │◇────────│ Student  │
└──────────┘          └──────────┘
  ◇ = 空心菱形 (学校没了，学生还在)
```

### 5. 组合 — 「整体-部分」强关系
```
┌──────────┐          ┌──────────┐
│   House  │◆────────│  Room    │
└──────────┘          └──────────┘
  ◆ = 实心菱形 (房子没了，房间也没了)
```
```python
class House:
    def __init__(self):
        self.rooms = [Room(), Room()]  # 同生共死
```

### 6. 依赖 — 「使用」关系
```
┌──────────┐         ┌──────────┐
│  Service │- - - - >│ Repository│
└──────────┘ 虚线箭头 └──────────┘
  方法参数/返回值中用到
```
```python
class Service:
    def save(self, repo: Repository):  # repo 作为参数
        repo.persist(self.data)
```

## 关系总结

| 关系 | 符号 | 代码 | 强度 |
|------|------|------|------|
| 泛化 | △ 实线 | `class A(B)` | 强 |
| 实现 | △ 虚线 | `class A(Interface)` | 强 |
| 组合 | ◆ 实线 | 成员变量，同生共死 | 强 |
| 聚合 | ◇ 实线 | 成员变量，独立存活 | 中 |
| 关联 | — 实线 | 成员变量 | 中 |
| 依赖 | ---> 虚线 | 方法参数/局部变量 | 弱 |
